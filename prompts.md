# 2. Prompt Utilizado
Contexto Académico: Atue como um Desenvolvedor Front-End. Preciso que crie uma aplicação Web (SPA) dividida estritamente em três arquivos: index.html, style.css e script.js. O projeto consiste numa tela de login que, após autenticação com sucesso, liberta o acesso a um CRUD de uma Lista de Filmes Assistidos. Restrições Rígidas de Sintaxe (Baseadas no Material de Aula): Organização das Funções: O código JavaScript não pode ter lógica solta e deve ser estruturado obrigatoriamente seguindo a arquitetura de funções nomeadas tradicionais: Funções de Coleta: Para capturar dados dos inputs do DOM. Funções de Validação: Para verificar campos vazios e credenciais, retornando erros ou booleanos. Funções de Exibição: Para renderizar os dados na tela e alternar a visibilidade das seções. Funções de Lógica/Orquestradoras: Para manipular o array e coordenar a execução das outras funções. Nota: Utilize a palavra-chave function tradicional (ex: function renderizarLista() {}). Evite funções anónimas ou inline. Manipulação de Listas e Loops: A base de dados deve ser um Array de Strings simples (proibido conter objetos). Para ler e exibir os itens do array no HTML, utilize obrigatoriamente um loop for tradicional (ex: for (let i = 0; i < lista.length; i++)). É estritamente proibido o uso de métodos avançados como .map(), .forEach() ou .filter(). Manipulação de Strings e Janelas (Apostila de Objetos e Introdução): Utilize o método .trim() para limpar espaços em branco nas validações. Para a criação do HTML dinâmico dentro do loop, utilize Template Strings obrigatoriamente com crase (). Utilize o método confirm() nativo para validar se o utilizador deseja mesmo remover um item ou cancelar a edição. Requisitos Funcionais da Aplicação: 1. Ecrã de Login: Inicialmente, apenas o formulário de login está visível. A área do CRUD deve estar oculta (display: none). Credenciais fixas: Utilizador: aluno | Palavra-passe: fiap2025. Se falhar ou os campos forem enviados vazios, exiba uma mensagem de erro numa div estilizada diretamente no ecrã (proibido usar alert ou console.log para erros de login). 2. Lista com CRUD Completo (Ecrã Pós-Login): A lista de strings deve iniciar com pelo menos 3 filmes predefinidos. Adicionar ao Final: Inserir o filme no fim do array (utilize .push()). Adicionar ao Início: Inserir o filme no início do array (utilize .unshift()). Editar: Botão individual que permite alterar o texto do filme com base na sua posição (índice i) no array. Se o utilizador confirmar o campo vazio ou cancelar (confirm()), o item original permanece intacto. Remover: Botão individual que remove o filme do array usando estritamente a sua posição (índice i) através do método .splice(index, 1), evitando apagar duplicados por valor. Atualização Automática: Qualquer alteração no array deve disparar a função orquestradora que limpa o campo e renderiza novamente a lista atualizada. 3. Validações de Campos: Nenhum filme pode ser adicionado ou editado se o campo estiver vazio (após o .trim()). Mostre um aviso caso aconteça. Output Esperado: Forneça o código limpo, comentado e separado para: index.html (Sem atributos de eventos inline como onclick="". Faça tudo via addEventListener no JS). style.css (Design moderno, limpo, simulando uma transição de ecrãs de forma fluida). script.js (Cumprindo rigorosamente a divisão de funções e restrições de loops mencionadas).
